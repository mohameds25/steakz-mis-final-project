# Frontend PRD - Steakz MIS

## Purpose

The frontend gives Steakz staff a responsive web dashboard for operating the restaurant chain according to their role.

## Primary Screens

| Screen | Purpose |
|---|---|
| Login | Demo role sign-in with JWT session storage. |
| Register | Customer account creation. |
| Overview | Shows branch count, open orders, revenue, and low-stock count. |
| Customer Portal | Lets customers choose a branch, place an order, and view own orders. |
| Inventory | Lists stock items and quantities. |
| Orders | Lists customer/table orders and preparation status. |
| Shifts | Lists staff schedules. |
| Branches | Lists restaurant branches. |

## Requirements

| ID | Requirement | Priority |
|---|---|---|
| FE-1 | Users can log in using backend credentials. | Must |
| FE-2 | Dashboard reflects authenticated role and branch. | Must |
| FE-3 | Show inventory, orders, shifts, sales metrics, and branches. | Must |
| FE-4 | Use responsive layout for laptop and mobile presentation. | Must |
| FE-5 | Display useful errors when API is unavailable. | Should |
| FE-6 | Use clear visual navigation and icons. | Should |
| FE-7 | Customers can register from the login screen. | Must |
| FE-8 | Customers see a simplified customer portal instead of staff MIS tools. | Must |
